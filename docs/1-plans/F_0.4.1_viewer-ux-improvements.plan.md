# Viewer UX Improvements

**Version target**: `0.4.1` (patch — UI-only, no engine/physics changes)
**Phase reference**: Pre-B4 UX polish for the assembly viewer and tree panel.

## Overview

Three viewer UX improvements matching modern CAD tool conventions (SolidWorks, FreeCAD): bidirectional hover-highlight sync between the 3D viewer and the assembly tree, per-part visibility toggles (eye icon) in the tree, and an RGB tri-axis orientation gizmo in the viewer corner.

## Problem Statement

- Identifying parts requires clicking — no hover preview linking the 3D view to the tree and vice versa.
- No way to hide parts to see internal geometry (PCBs, nested components).
- No spatial orientation reference in the viewer — users can't tell which axis is which after orbiting.

## Solution Architecture

All three features are UI-only — no changes to the engine, schemas, deterministic output, or the geometry pipeline.

1. **Hover-highlight sync**: pythonocc's `qtViewer3d` already calls `display.MoveTo(x, y)` on mouse move, which drives OCCT's dynamic highlighting. After `MoveTo`, query `Context.DetectedInteractive()` to find which `AIS_Shape` is under the cursor, resolve it to a `solid_id`, and emit a signal. The tree panel listens and scrolls to + temporarily highlights the corresponding row. Conversely, the tree emits a hover signal when the mouse enters a row; the viewer calls `Context.SetHighlighted()` on the corresponding `AIS_Shape`.

2. **Per-part visibility toggle**: Add a 5th column to the tree header with an eye icon. Each item stores visibility state. Clicking the eye calls `Context.Erase(ais)` or `Context.Display(ais)`. Toggling a parent node propagates to all children.

3. **Tri-axis gizmo**: Use OCCT's `V3d_View.TriedronDisplay()` which renders a small XYZ triad in a corner of the view. Configurable position, size, and colors (Red=X, Green=Y, Blue=Z). One call in `init_viewer`, no ongoing maintenance.

## Implementation Details

### 1. Hover-highlight: Viewer → Tree

**File**: `src/raysim/ui/viewer.py` (modify)

Add a new signal `shape_hovered = Signal(str)` on `ViewerWidget`.

Add a `_setup_hover_detection()` method called from `init_viewer()`. This installs a periodic timer (e.g. 50ms) that, after each `MoveTo` (already happening in `qtViewer3d.mouseMoveEvent`), queries `Context.HasDetected()` and `Context.DetectedInteractive()`. If the detected `AIS_Shape` maps to a `solid_id`, emit `shape_hovered`. Track the last-emitted ID to avoid signal spam.

Alternative to a timer: subclass `qtViewer3d` and override `mouseMoveEvent` to call our detection after `super().mouseMoveEvent`. This is more direct but couples to pythonocc's internal class. The timer approach is less coupled.

**Recommended approach**: Install an event filter on `self._viewer` via `installEventFilter`. Qt event filters fire *before* the widget processes the event, so the detection query must be deferred via `QTimer.singleShot(0, self._detect_hover)` to run after `qtViewer3d`'s own `mouseMoveEvent` has called `display.MoveTo()`. This avoids subclassing pythonocc internals while ensuring the OCCT context is up-to-date when we query `DetectedInteractive()`.

### 2. Hover-highlight: Tree → Viewer

**File**: `src/raysim/ui/panels/tree_panel.py` (modify)

Enable mouse tracking on the `QTreeWidget` (`setMouseTracking(True)`). Override or connect `itemEntered` signal (requires `setMouseTracking(True)`). When a tree item is entered, extract the `solid_id` from column 2 and call `viewer.highlight_solid(solid_id)`. When the mouse leaves the tree area, clear the highlight.

To avoid conflicting with click-based selection, use OCCT's `HilightWithColor` or `SetHighlighted` for hover (temporary visual feedback) vs `AddOrRemoveSelected` for click (persistent selection).

Add a `highlight_hovered(solid_id)` method on `ViewerWidget` that uses `Context.HilightWithColor(ais, style, True)` for temporary highlighting, distinct from `highlight_solid` which uses `AddOrRemoveSelected`.

### 3. Hover-highlight: Viewer → Tree (tree side)

**File**: `src/raysim/ui/panels/tree_panel.py` (modify)

Connect `ViewerWidget.shape_hovered` to a new `_on_viewer_hover` slot. This scrolls to and temporarily highlights (e.g. background color) the corresponding tree row without changing the current selection. Use `QTreeWidget.scrollToItem()` with `EnsureVisible` hint.

Clear the temporary highlight when the viewer emits a hover with an empty string (mouse moved off all shapes).

### 4. Per-part visibility toggle (eye icon)

**File**: `src/raysim/ui/panels/tree_panel.py` (modify)

Add a 5th column header: `""` (icon-only, narrow). Set column width to ~24px via `setColumnWidth(4, 24)`.

In `_add_node`, set each item's column-4 icon to an open-eye icon. Store visibility state as item data (e.g. `Qt.ItemDataRole.UserRole`).

Connect `itemClicked` to a handler that checks if column 4 was clicked. On click: toggle visibility state, swap icon (open ↔ closed eye), call `viewer.set_solid_visible(solid_id, visible)`.

For parent nodes (non-leaf), toggling propagates to all descendant leaf items recursively.

**File**: `src/raysim/ui/viewer.py` (modify)

Add `set_solid_visible(solid_id, visible)`:
- `visible=True`: `context.Display(ais, False)` + `context.UpdateCurrentViewer()`
- `visible=False`: `context.Erase(ais, False)` + `context.UpdateCurrentViewer()`

Batch the `UpdateCurrentViewer()` call when propagating to children (call once after all children are toggled, not per-child).

**Icon assets**: Use simple Unicode characters as text or create two small 16x16 PNG icons under `src/raysim/ui/icons/`. Unicode is simpler and cross-platform. Alternatively, use `QStyle.StandardPixmap` for a built-in icon that doesn't require asset files.

### 5. Tri-axis orientation gizmo

**File**: `src/raysim/ui/viewer.py` (modify)

In `init_viewer()`, after `InitDriver()`, add:

```
view = self._display.View
view.TriedronDisplay(
    Aspect_TOTP_LEFT_LOWER,   # position: lower-left corner
    Quantity_NOC_RED,          # X axis color
    0.1,                       # scale (fraction of view)
    V3d_WIREFRAME,             # display mode
)
```

The OCCT `V3d_View.TriedronDisplay` method renders a small axis triad that rotates with the camera automatically. The colors for Y and Z are set via `Quantity_NOC_GREEN` and `Quantity_NOC_BLUE` through the `V3d_TypeOfAxe` parameters if the overload is available; otherwise the default triad uses distinct axis colors.

Check the pythonocc binding signature: `V3d_View.TriedronDisplay(position, color, scale, mode)`. The single-color overload sets all three axes to one color; the `Graphic3d_GraduatedTrihedron` approach (already used in pythonocc's `OCCViewer.py`) offers per-axis color control.

**Fallback**: If `TriedronDisplay` doesn't support per-axis colors, use `Graphic3d_GraduatedTrihedron` (already used in pythonocc's `OCCViewer.py`) which renders in screen-space and supports per-axis configuration. Do NOT use `AIS_Trihedron` for this purpose — it lives in world-space and cannot be pinned to a viewer corner.

## Technical Considerations

- **Determinism Impact**: None. UI-only changes, no effect on `run.json`, provenance, or canonical JSON.
- **Numerical Precision**: Not applicable.
- **Stack Accumulator Invariants**: Not applicable.
- **Material Physics Scope**: Not applicable.
- **Coincident-Face / Tied-Batch**: Not applicable.
- **Backend Optionality**: All changes are inside `raysim.ui` which already requires PySide6 + pythonocc-core. No new optional deps.
- **Schema Versioning**: No schema changes.
- **Performance**: Hover detection runs on every mouse move but the OCCT context already tracks the detected shape internally via `MoveTo` — querying `DetectedInteractive()` is a dict lookup, not a re-trace. No measurable overhead expected.
