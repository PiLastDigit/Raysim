# Decouple Overlap Diagnostic from Pipeline

**Version target**: `0.4.0` (minor — gate relaxation + new validate command + pipeline optimization)
**Phase reference**: Addresses B1.5/B1.6 performance bottleneck surfaced
during B3 UI testing on real assemblies (250+ solids).

## Overview

Remove the slow O(N²) volume-intersection classification from the mandatory
STEP→Embree pipeline. Keep the fast triangle-match tied-group extraction
and mismatched-contact detection inline. Add an optional on-demand
"Validate Geometry" action (UI + CLI) for the full interference diagnostic.
Phase 2 adds spatial-hash pair filtering to make both paths faster.

## Problem Statement

`build_assembly_from_step()` calls `diagnose_overlaps()` which runs an
O(N²) loop over all solid pairs. For each AABB-overlapping pair, it
performs both triangle-level vertex-match tied pairing (fast) and
OCCT volume intersection classification (slow). On a 250-solid PCB,
this takes minutes — freezing the UI on the "Run" action.

The meca team validates CAD designs before import, so interference is
rare and minimal. The volume classification should be an optional sanity
check, not a mandatory pipeline gate.

## Solution Architecture

Split `diagnose_overlaps` into two tiers:

1. **Fast contact extraction** (`extract_contacts`) — AABB filter +
   `_detect_coplanar_contacts` triangle matching. Returns a
   `ContactReport` carrying `tied_pairs` + `mismatched_contacts`.
   No OCCT volume classification. Stays in the mandatory pipeline
   because the ray engine needs tied groups (ARCHI §11) and
   mismatched contacts must still gate/warn (ARCHI §9b).

2. **Full overlap diagnostic** (`diagnose_overlaps`, unchanged) —
   calls `extract_contacts` + the O(N²) volume classification.
   Moved to an on-demand path: UI "Validate Geometry" button and
   `raysim validate` CLI subcommand.

**Intentional gate relaxation**: This change downgrades the full
interference diagnostic from a mandatory pipeline gate to an advisory
on-demand check. The MVP_PLAN §7 requirement ("overlap/interference
diagnostic clean or accepted in the project file") is satisfied by
the fast-path mismatched-contact gate (still mandatory) plus user-
initiated validation when desired. The rationale: the meca team
validates CAD designs before import, so interference is rare and
minimal. The full O(N²) volume check is a sanity tool, not a safety
gate. The `interference_overrides` field in the project file continues
to persist user override decisions from validation results when the
user chooses to run validation. ARCHI.md, MVP_PLAN.md §7, and MVP_STEPS.md will be
updated to reflect this relaxation.

## Implementation Details

### 1. Add `extract_contacts()` and `ContactReport` to overlap.py
**File**: `src/raysim/geom/overlap.py` (modify)

New frozen dataclass:
```python
@dataclass(frozen=True)
class ContactReport:
    tied_pairs: tuple[TiedTrianglePair, ...]
    mismatched_contacts: tuple[MismatchedContactRegion, ...]
```

New public function `extract_contacts(solids, *, tolerances...)`:
- AABB overlap filter per pair
- `_detect_coplanar_contacts` per overlapping pair
- Returns `ContactReport` (no `OverlapPair`, no classification)

Refactor `diagnose_overlaps` to call `extract_contacts` internally,
then layer on `_classify_pair` for each AABB-overlapping pair. No
behavioral change to `diagnose_overlaps` itself.

### 2. Update `ValidatedAssembly` in pipeline.py
**File**: `src/raysim/geom/pipeline.py` (modify)

**Current `ValidatedAssembly` fields:**
```python
solids, watertightness, overlaps: OverlapReport, overrides_used
```

**New fields:**
```python
solids, watertightness,
contacts: ContactReport,              # fast path (always populated)
overlaps: OverlapReport | None = None, # full diagnostic (on-demand)
overrides_used
```

`build_assembly_from_step()` changes:
- Replace `diagnose_overlaps(healed, shapes=shape_map)` with
  `extract_contacts(healed)` (around line 75-76)
- The `shape_map` is no longer computed at pipeline time

### 3. Update adapter.py for ContactReport
**File**: `src/raysim/geom/adapter.py` (modify)

`_build_tied_groups` reads from `assembly.contacts.tied_pairs` instead
of `assembly.overlaps.all_tied_triangle_pairs()`.

`_gate_overlaps` is split:
- **Mismatched-contact gate** (always active): warns or blocks on
  `assembly.contacts.mismatched_contacts` unless `accept_warnings=True`
- **Interference gate** (only when overlaps populated): checks
  `assembly.overlaps.failed()` / `.warnings()` when not `None`;
  is a no-op when `assembly.overlaps is None`

### 4. Add `raysim validate` CLI subcommand
**File**: `src/raysim/cli/main.py` (modify)

```
raysim validate --step PATH [--accept-warnings] [--json-out PATH]
```

**Exact flow:**
1. `load_step(path)` → `iter_leaves` → leaf list
2. `tessellate()` per leaf
3. `heal_assembly(tessellated)` → healed solids (triangle-level)
4. `validate_watertightness(healed)` — report only, no gate
5. `shape_map = {leaf.solid_id: leaf.shape for leaf in leaves}`
   (original `TopoDS_Solid` from `load_step`, used for OCCT boolean
   volume classification — same as the current `pipeline.py` line 75)
6. `diagnose_overlaps(healed, shapes=shape_map)` — full diagnostic
7. Print summary to stdout; write full JSON if `--json-out` given

**JSON serialization** (`--json-out`): `dataclasses.asdict()` on the
`OverlapReport` → `json.dumps(indent=2)`. This is a human-readable
diagnostic artifact, not a canonical-JSON deterministic stream.

**Exit codes:**
- `0`: clean (all pairs `contact_only` or `accepted_nested`)
- `1`: `interference_fail` or `interference_warning` detected
  (unless `--accept-warnings` downgrades warnings to exit 0)
- `2`: usage error (click standard)

### 7. Provenance traceability
**File**: `src/raysim/ui/workers/run_worker.py` (modify)

Validation status goes in the **human-metadata sibling only** (the
`--human-metadata-out` file), not in the deterministic `run.json`
`Provenance` block. No `SCHEMA_VERSION` bump. Validation doesn't
change the physics answer — it's a confidence-level annotation.

The human-metadata block gets:
- `overlap_validation: "validated"` or `"skipped"`
- `overlap_report_summary: {n_contact, n_nested, n_warning, n_fail}`
  when validated

**Path convention for UI runs**: `RunContext` gains an optional
`human_metadata_path: Path | None` field. When `output_path` is set,
`human_metadata_path` defaults to `output_path.with_suffix('.human.json')`.
The CLI already has `--human-metadata-out`; the UI auto-derives it.

### 8. Phase 2 — Pair-filtering optimization
**File**: `src/raysim/geom/overlap.py` (modify)

Three complementary optimizations. Spatial hash applies to both
`extract_contacts` and `diagnose_overlaps`. Negative margin and volume
early-exit apply only to `diagnose_overlaps` (volume classification).
All are internal — the public API is unchanged.

**a. Spatial-hash pair filtering.** Replace the O(N²) double loop with
a grid-based spatial hash:
1. Compute grid cell size = `median(solid_bbox_diag) × 2`
2. Hash each solid's AABB into the grid cells it overlaps
3. Only check pairs that share at least one cell
4. Canonical ordering (`solid_a < solid_b`) deduplicates

Expected reduction on 250-solid PCB: ~31,000 pairs → ~500 neighbors.

**b. AABB negative-margin filter.** Tighten `_aabb_overlap` for the
volume-classification path only (not `extract_contacts`). Coincident
faces have merely touching AABBs — shrinking them would skip exactly
the pairs needed for tied groups. The negative margin applies only
inside `diagnose_overlaps` when filtering candidate pairs for the
expensive `_classify_pair` OCCT volume check, after `extract_contacts`
has already collected all tied pairs and mismatched contacts.

**c. Early-exit on volume classification.** In `_classify_pair`, skip
`BRepAlgoAPI_Common` only when BOTH conditions hold: (a) the contact
report shows full-coverage contact with no mismatches, AND (b) the
negative-margin AABB filter says the pair has no positive-volume
intersection candidate. If the pair passes the margin filter (potential
volume overlap), always run `_classify_pair` — two solids can touch at
one face and interfere at another. Triangle enumeration within
`_detect_coplanar_contacts` is never skipped — all face pairs are
checked to collect every tied triangle and mismatched region.

## Technical Considerations

- **Determinism Impact**: None. `run.json` output is unchanged. The
  tied-group pairs from `extract_contacts` are structurally identical
  to what `diagnose_overlaps` produced. The overlap validation status
  is in human-metadata only, not in the deterministic stream.

- **Numerical Precision**: No impact. Same vertex-match tolerances.

- **Coincident-Face / Tied-Batch**: `extract_contacts` preserves the
  exact same `TiedTrianglePair` records. The adapter's `_build_tied_groups`
  receives identical input. Ray engine invariants (ARCHI §11) unaffected.

- **Material Physics Scope**: Unchanged.

- **Backend Optionality**: `extract_contacts` is pure numpy — no OCCT
  needed. `diagnose_overlaps` with `shapes` needs OCCT for volume
  classification. The `raysim validate` CLI guards OCCT at call time.

- **Schema Versioning**: No `SCHEMA_VERSION` bump for `run.json`.
  No `PROJECT_SCHEMA_VERSION` bump — the project file's
  `interference_overrides` field already exists and is sufficient for
  persisting user override decisions from validation results.

- **Edge Cases**:
  - Assembly with 0-1 solids: both functions return empty results
  - Assembly with no AABB overlaps: fast path skips all pairs
  - novtk build: `extract_contacts` works; `diagnose_overlaps` with
    `shapes=None` does triangle-level only (no volume classification)
  - User runs without validating: `run.json` is correct, human-metadata
    notes validation was skipped

## Performance & Cost Impact

**Phase 1 (decouple)**:
- Pipeline on 250-solid PCB: minutes → seconds (drops `_classify_pair`)
- `extract_contacts` on same PCB: ~1-5s (AABB filter + triangle match)
- On-demand `diagnose_overlaps`: unchanged (minutes), but user-triggered

**Phase 2 (pair-filtering optimization)**:
- Spatial hash (both paths): ~31,000 pairs → ~500 relevant neighbors
- AABB negative margin (`diagnose_overlaps` only): eliminates volume
  classification candidates that are close but not intersecting
- Volume early-exit (`diagnose_overlaps` only): contact-only pairs
  skip redundant OCCT volume check after full triangle enumeration
- Combined: `extract_contacts` ~0.1-0.5s, `diagnose_overlaps` minutes → seconds

## Backward Compatibility

- `ValidatedAssembly.overlaps` becomes optional (`None` by default).
  `ValidatedAssembly` gains `contacts: ContactReport` (always populated).
- Adapter gate split: mismatched contacts still warn/block;
  interference gate only fires when `overlaps` is populated.
- No `run.json` schema change. No project file schema change.
  `interference_overrides` in the project file continues to work.
- Existing tests: `test_overlap.py` and `test_geom_adapter.py` need
  updates for the new `ContactReport` field and optional `overlaps`.
