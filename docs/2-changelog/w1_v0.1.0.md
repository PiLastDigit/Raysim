## Changes

Phase B1 adds the `raysim.geom` package — the geometry pipeline that turns a single STEP file into the same `BuiltScene` the Stage A engine consumes. The pipeline is:

- **B1.1 `step_loader`**: Reads STEP via `STEPControl_Reader`, walks the compound recursively into `LeafSolid` records with synthetic IDs (`solid_NNNN`) and proper path keys.
- **B1.2 `tessellation`**: Runs `BRepMesh_IncrementalMesh` per leaf, extracts per-shell triangle arrays in float64 with location transforms applied.
- **B1.3 `healing`**: `BRepMesh_ModelHealer` pass + shell-orientation normalization via per-shell probe rays. Classifies each shell as OUTER or CAVITY. Full entry/exit stack-to-zero re-verification after flips.
- **B1.4 `watertightness`**: Per-shell edge-pairing validator with vertex canonicalization (1e-9 mm tolerance). Reports unpaired edges, same-orientation edges, and degenerate triangles.
- **B1.5 `overlap`**: Triangle-level vertex-match tied pairing + four-way solid-pair classification (contact_only / accepted_nested / interference_warning / interference_fail). Mismatched-tessellation contacts detected and gated. OCCT boolean volume classification via `BRepAlgoAPI_Common`.
- **B1.6 `pipeline` + `adapter`**: Orchestrator gluing B1.1→B1.5. Adapter exports deterministic binary STLs (lex-sorted, in-house writer), translates tied groups through index maps into `PreBuiltTiedGroups`, and hands off to `load_scene_from_directory`. Validation gates with three-flag override scheme.

`raysim.ray.scene` gains an additive `PreBuiltTiedGroups` type and `tied_groups` / `process_meshes` parameters. Stage A's STL path is unchanged.

All B1 modules guard OCC imports at call time; `pytest.importorskip("OCC.Core")` skips tests without pythonocc-core. STEP fixture generator script at `scripts/build_step_fixtures.py` (requires OCCT conda env to run).
