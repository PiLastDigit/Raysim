# Changelog - Week 1, 30-04-2026, V. 0.2.0

**Release Date**: Week 1, 30-04-2026 at 13:29
**Version**: 0.2.0 (previously 0.1.0)
**Object**: Add Phase B2 materials + project file: library, rules, STEP tags, review, gating, .raysim format
**Code review**: `docs/3-code-review/CR_w1_v0.2.0.md` (Codex loop, 3 rounds, APPROVED with observations)

## Changes

- **New package**: `raysim.mat` — seeded material library (14 entries: Al 6061,
  Cu, Si, SiO2, Kapton, FR4, Ti-6Al-4V, W, GaAs, Au, Sn-Pb solder, battery,
  populated PCB, harness), naming-rule auto-assignment engine
  (regex-to-group_id YAML), STEP material-tag ingestion (OCCT-optional with
  two-gate verification: leaf count + bbox fingerprint), auto-assignment
  review API (manual > step_tag > naming_rule priority), and run gating with
  density anomaly warnings.
- **New module**: `raysim.proj.project` — `.raysim` project file schema
  (Pydantic, canonical JSON, bit-stable save/reopen/save), geometry ref with
  path + SHA-256, naming-rule overrides, interference overrides, schema
  version with N-1 acceptance.
- **New type**: `SolidRef` — stable input token for rules and review APIs,
  carrying `solid_id`, `path_key`, and `display_name`.
- **Default data files**: `default_library.yaml` (14 NIST/vendor-sourced
  materials), `default_rules.yaml` (16 regex patterns covering common
  spacecraft naming conventions).
- **Tests**: 6 new test files (47 tests), all pure-Python except one
  OCCT-gated integration test.
- **Bug fixes during review**: blank STEP tags no longer auto-assign;
  invalid manual assignments no longer fall through to auto-sources;
  unresolved manual assignments now appear in the CLI review summary.
