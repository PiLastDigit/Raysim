# Changelog - Week 1, 01-05-2026, V. 0.3.1

**Release Date**: Week 1, 01-05-2026 at 18:04
**Version**: 0.3.1 (previously 0.3.0)
**Object**: Fix XCAF name extraction, add instance/prototype name columns to assembly tree
**Code review**: `docs/3-code-review/CR_w1_v0.3.1.md` (Codex loop, 2 rounds, APPROVED)

## Changes

- Fixed XCAF label name extraction: `_get_label_name` now uses `TDF_Label.GetLabelName()` instead of the broken `TDataStd_Name.FindAttribute` path (SWIG type mismatch silently caught by bare except).
- XCAF document creation uses `TDocStd_Document("pythonocc-doc-step-import")` (pythonocc's own pattern from `DataExchange.py`) instead of `TDocStd_Application` + `BinXCAFDrivers`.
- Separated instance names (component/reference label, e.g. "R41") from prototype names (referred shape label, e.g. "R_0402_1005Metric") — instance name takes priority for `LeafSolid.name`, prototype stored in new `LeafSolid.part_name` field.
- Assembly tree panel now shows 4 columns: Name (instance), Part (prototype), Solid ID, Material.
- `SolidRef` gains `part_name` field; naming rules now match against `solid_id`, `path_key`, `display_name`, and `part_name`.
